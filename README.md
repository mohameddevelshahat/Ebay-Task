# Ebay Automation Testing Project

## ðŸ“Œ Overview
This project contains automated tests using **Selenium WebDriver**, **Java**, and **TestNG**.
The framework follows the **Page Object Model (POM)** design pattern and includes **Allure Report** for test execution results.

**Features** :-

1- Navigate to eBay
and validate landing on the homepage.

2- Perform a search for a specific item (e.g., "mazda mx-5").

3- Validate that search results are displayed.

4- Print/log the number of obtained search results.

5- Apply a left-hand side filter (Free International Shipping -> Manual).

6- Framework supports Page Object Model (POM) for better organization.

7- External test data is read from JSON.

8- Execution reports generated using TestNG and Allure report.

9- Handles common browser actions like maximize window, click, input text, etc.

**Prerequisites** :-

1- Java JDK 11 or higher

2- Maven

3- IDE (IntelliJ IDEA)

4- Chrome browser + ChromeDriver + Firfox browser and Edge prowser

5- Basic knowledge of Selenium WebDriver, TestNG, and Java OOP

**Test Data** :-

The framework reads test data from an external JSON file:

**How to Run**

1- Clone the repository

2- Open the project in your preferred IDE

3- Ensure ChromeDriver path is set correctly in DriverManager.java

4- Compile all Java classes

5- Run the tests using TestNG (executor.xml)

6- Test execution reports will be generated in the test-output folder

7- Run allure report

Page Object Model (POM)

1- Home_Page.java: Represents the eBay homepage. Contains methods for opening the homepage and searching for items

2- Search_ResultsPage.java: Represents the search results page. Contains methods for validating results, counting results, and applying filters

Utilities

DataUtils: Reading data from JSON and Properites Files

LogsUtils: To Log and trace code 

Utility: Dealing with Actions needed to the project
